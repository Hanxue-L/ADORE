# Machine-Learning Workflow

The workflow uses paired DPV features as an eight-dimensional input vector.

```text
DPV traces
→ feature extraction
→ paired feature table
→ classifier training
→ cross-validated metric summary
```

Run classification with:

```bash
python scripts/train_classifier.py --input path/to/feature_table.csv --models svm baseline rf knn
```

Available model names:

| Name | Model |
| --- | --- |
| `baseline` | Logistic regression baseline using `miR92a_Delta_I` |
| `svm` | RBF-kernel support vector machine |
| `rf` | Random forest |
| `knn` | K-nearest neighbors |
| `mlp` | Multilayer perceptron |

The script performs 5 repeats of stratified 5-fold cross-validation by default. For each outer fold, hyperparameters are selected with inner 3-fold cross-validation and Optuna TPE search. Preprocessing and model fitting are performed inside a scikit-learn pipeline. Threshold-based metrics use a decision threshold selected from the training portion of the fold and applied to the held-out portion.

The threshold is used only for cross-validated metric reporting. It is selected from probabilities produced on the outer-fold training data, so threshold-dependent metrics should be interpreted as operating-point estimates rather than as a separately locked deployment threshold. The fitted SVM prediction script uses the model's default decision boundary (`model.predict()`), not the Youden threshold used for CV metric reporting.

Apply the fitted SVM model to a compatible feature table:

```bash
python scripts/predict_with_svm_model.py --input path/to/feature_table.csv --output results/svm_predictions.csv
```

The fitted model uses the feature order documented in `models/model_metadata.json`. Class assignments are generated by the fitted SVM decision boundary, and `probability_disease` reports the class-1 probability. For SVC models, the predicted class and probability are generated by different mechanisms and may not agree at a 0.5 probability cutoff.
