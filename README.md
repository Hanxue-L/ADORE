# ADORE

This repository provides the DPV parsing, electrochemical feature extraction, SVM model, and classification-evaluation workflow used in the ADORE analysis.

## Repository Scope

This repository includes the DPV parsing and feature-extraction code, paired feature-table construction, the classification/evaluation workflow, a fitted SVM model (`models/adore_svm_model.joblib`) with metadata, representative example traces, and the test suite.

The fitted SVM is provided as `models/adore_svm_model.joblib`; `models/model_metadata.json` records its hyperparameters, feature order, class labels, scikit-learn version, and SHA-256 checksum for integrity checking.

## DPV Feature Extraction

Run the example extraction workflow from the repository root:

```bash
python scripts/extract_features_from_examples.py
```

The script reads two representative DPV traces:

- `examples/raw_dpv/mb_example.txt`
- `examples/raw_dpv/l012_example.txt`

It writes:

```text
examples/example_features.csv
```

Each trace is converted into four features:

- `Delta_I`
- `Ep`
- `Ah`
- `FWHM`

The DPV peak current `ip` is extracted from the valley-to-valley baseline-corrected trace. `Delta_I` is calculated as `blank_current - ip`; set `blank_current` in `examples/dpv_example_manifest.csv` according to the blank or control current measured in the assay. The `current_scale` column converts raw current values into the same current unit used for `blank_current`.

Input traces may contain a `voltage,current` or `Potential/V` header, or begin directly with numeric voltage/current pairs. Voltage values must be strictly monotonic, and the extracted upward peak must have a valley on each side.

Build the paired feature-table format used by the classifier:

```bash
python scripts/build_feature_table.py
```

The script writes:

```text
examples/example_feature_table.csv
```

## Classification Workflow

The classifier script accepts a feature table with paired miR-21 and miR-92a features:

```bash
python scripts/train_classifier.py --input path/to/feature_table.csv --models svm baseline rf knn
```

The input schema is documented in:

```text
docs/feature_table_schema.md
```

The table template is:

```text
examples/feature_table_template.csv
```

## Fitted SVM Model

The fitted SVM model is provided in:

```text
models/adore_svm_model.joblib
```

Model metadata, class labels, and feature order are documented in:

```text
models/model_metadata.json
```

Run prediction with:

```bash
python scripts/predict_with_svm_model.py --input path/to/feature_table.csv --output results/svm_predictions.csv
```

The predicted class is assigned by the fitted SVM decision boundary using `model.predict()`. The output also reports the class-1 probability as `probability_disease` using `model.predict_proba()`. For scikit-learn SVC models, these two outputs come from different mechanisms, so `probability_disease` does not need to be above 0.5 for `predicted_class` to be `Disease`.

The cross-validation metrics generated by `scripts/train_classifier.py` use a Youden-thresholded operating point selected within each outer training fold. This operating point is distinct from the default decision boundary used by `scripts/predict_with_svm_model.py`.

## Models

The training workflow includes:

- SVM
- Logistic regression baseline
- Random forest
- K-nearest neighbors
- Multilayer perceptron

The training workflow uses nested repeated cross-validation. Standardization is performed inside each scikit-learn pipeline.
